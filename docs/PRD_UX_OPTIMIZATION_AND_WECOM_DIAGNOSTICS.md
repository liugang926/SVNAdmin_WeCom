# PRD: 仓库权限体验、布局响应式与 WeCom 一键诊断优化

## 1. 文档信息

- 产品名称: SVNAdmin WeCom Edition
- 需求类型: 体验优化 / 运维诊断增强
- 目标版本: vNext
- 需求来源: 管理员在仓库权限维护、复杂表格浏览、后台任务查看、企业微信排障中的效率与可用性问题
- 相关模块:
  - 前端仓库页: `01.web/src/views/repositoryInfo/index.vue`
  - 权限配置弹窗: `01.web/src/components/modalRepPri.vue`
  - 全局布局: `01.web/src/views/layout/basicLayout/index.vue`
  - 企业微信页: `01.web/src/views/wecom/index.vue`
  - WeCom 配置组件: `01.web/src/views/wecom/components/WecomConfig.vue`
  - WeCom 后端控制器: `02.php/app/controller/WeComAdmin.php`

## 2. 背景与问题

当前项目已经具备 SVN 仓库管理、权限配置、后台任务、企业微信配置/同步/通知/映射/监控等核心能力，但在高频管理场景中仍存在几类体验摩擦:

1. 仓库数量、权限路径增多后，`repositoryInfo` 页面仍以手动搜索和表格浏览为主，定位仓库与权限路径的效率不足。
2. `global-task-ball` 作为全局悬浮入口与主业务界面视觉层级接近，且固定悬浮在右下角，容易遮挡表格操作区或关键按钮。
3. 全局 `Layout` 侧边栏固定 200px，在仓库列表、用户映射、通知规则等复杂表格场景下挤占横向空间。
4. WeCom 页面当前可显示整体状态，并已有连接测试类接口能力，但缺少面向管理员的一键诊断闭环。用户看到异常后仍需要自行判断 PHP 扩展、配置项、网络连通性、企业微信 API 权限、数据库表等问题。

## 3. 产品目标

1. 将仓库与权限路径定位从“搜索后刷新列表”升级为“输入即联想、快速命中、可继续追踪权限层级”。
2. 让后台任务入口从业务操作区中视觉脱离，降低遮挡核心操作按钮的概率。
3. 通过侧边栏折叠模式为复杂表格释放横向空间，并保持导航可识别。
4. 将 WeCom 状态展示升级为“一键诊断 + 交互式排错指引”，让管理员能直接按步骤修复常见问题。

## 4. 用户与场景

- 系统管理员: 管理仓库、权限、用户、后台任务和 WeCom 集成配置。
- 项目管理员: 关注指定仓库权限路径、成员访问范围和仓库列表定位。
- 运维人员: 关注 PHP 运行环境、网络连通性、数据库表、企业微信 API 凭证和同步任务状态。

典型场景:

1. 管理员输入仓库名片段，希望不离开当前页面即可看到匹配仓库并跳转到操作行。
2. 管理员在权限弹窗中面对多级路径，希望清楚看到父子层级、继承关系和当前授权主体。
3. 管理员处理宽表格时，希望暂时折叠左侧导航以查看更多列。
4. WeCom 同步失败后，运维希望点击一次诊断，看到失败节点、原因、建议操作和可复制的排查信息。

## 5. 需求范围

### 5.1 In Scope

- `repositoryInfo` 页面新增 Search-as-you-type 联想搜索。
- 权限路径展示层级优化，优先覆盖仓库权限配置与用户权限查看相关弹窗。
- `global-task-ball` 视觉层级与位置策略优化。
- 全局 `Layout` 侧边栏新增展开/折叠模式。
- WeCom 页面新增“一键诊断”入口、诊断结果面板和交互式排错指引。
- 后端新增或扩展 WeCom 诊断接口，输出结构化诊断项。

### 5.2 Out of Scope

- 不重构 SVN 权限模型和 authz 存储格式。
- 不改变现有角色权限体系。
- 不引入新的前端 UI 框架。
- 不在本期实现自动修复生产环境配置，仅提供明确指引和必要的安全操作入口。

## 6. 功能需求

### 6.1 仓库页 Search-as-you-type 联想搜索

#### 6.1.1 入口与交互

- 在 `repositoryInfo` 页面现有搜索框基础上增强为联想搜索。
- 用户输入 1 个字符后开始展示本地候选；输入 2 个及以上字符时支持远程候选或当前分页外候选。
- 候选项展示仓库名、备注、版本号、体积、最近更新时间或可用状态。
- 支持键盘操作: 上下键切换、Enter 选中、Esc 关闭。
- 选中候选后:
  - 若仓库已在当前表格中，滚动并高亮对应行 2 秒。
  - 若仓库不在当前页，自动执行搜索并定位到结果。
  - 保留当前角色下可见范围，普通用户只能联想其有权限的仓库。

#### 6.1.2 搜索规则

- 支持按仓库名、备注搜索。
- 匹配优先级: 完全匹配 > 前缀匹配 > 包含匹配 > 备注匹配。
- 候选数量默认最多 10 条。
- 输入防抖建议 250-300ms。
- 加载中展示轻量状态，空结果展示“未找到匹配仓库”。

#### 6.1.3 验收标准

- 输入仓库名前缀后 300ms 左右出现候选。
- 1000 个仓库以内联想不卡顿，候选响应主观无明显等待。
- 选中候选后能准确定位或刷新到目标仓库。
- 普通用户不会看到无权限仓库候选。

### 6.2 权限路径展示层级优化

#### 6.2.1 展示目标

- 将权限路径从平铺文本升级为层级化路径树。
- 路径层级需表达:
  - 根路径 `/`
  - 子目录层级
  - 当前路径是否配置了显式权限
  - 是否继承父级权限
  - 授权主体类型: 用户 / 用户组 / WeCom 同步组

#### 6.2.2 交互要求

- 权限弹窗左侧使用路径树或可展开列表，右侧展示选中路径的权限详情。
- 路径节点支持搜索过滤，匹配节点展开其父级。
- 支持快速筛选:
  - 仅看有显式权限的路径
  - 仅看当前用户/组相关路径
  - 仅看无权限或继承异常路径
- 权限标识建议:
  - 读写: `RW`
  - 只读: `R`
  - 无权限: `-`
  - 继承: 使用弱化标签
- 长路径需中间省略，但 hover 或详情区可查看完整路径。

#### 6.2.3 验收标准

- 管理员可以在 3 次点击内定位到任一已配置权限路径。
- 权限路径树能正确体现父子关系。
- 对超过 5 层、超过 80 字符的路径仍保持可读。
- 不改变现有权限保存结果。

### 6.3 后台任务悬浮球视觉分层与避让

#### 6.3.1 视觉分层

- `global-task-ball` 使用与主业务卡片不同的毛玻璃样式:
  - 半透明深色或中性色背景
  - 更强背景模糊
  - 独立阴影与描边
  - 运行中状态使用轻量动效和状态色描边
- 文案与图标聚焦“后台任务”，避免与业务按钮风格混淆。

#### 6.3.2 位置与遮挡策略

- 默认位置从右下角核心操作区上移，建议为 `right: 24px; bottom: 96px`。
- 当页面存在表格底部分页、固定操作按钮或弹窗时，悬浮球应避免遮挡:
  - 支持拖拽并记忆位置。
  - 支持收起为窄条或最小化图标。
  - 移动端默认放置在右上或底部安全区域，避开主操作按钮。
- 鼠标悬停时展开任务摘要，离开后收起。

#### 6.3.3 验收标准

- 仓库页、用户映射页、通知规则页的主要操作按钮不被默认悬浮球遮挡。
- 运行中、空闲、异常三种状态视觉可区分。
- 拖拽后刷新页面仍保留位置。

### 6.4 Layout 侧边栏折叠模式

#### 6.4.1 基础能力

- 全局侧边栏支持展开/折叠。
- 展开宽度保持 200px。
- 折叠宽度建议 56-64px，仅展示图标。
- Header 或侧边栏顶部增加折叠按钮。
- 折叠状态需本地持久化，刷新后保持。

#### 6.4.2 导航体验

- 折叠时:
  - 菜单组标题隐藏或转为分隔线。
  - 菜单项仅展示图标，hover 展示 Tooltip。
  - 当前路由高亮保留。
  - WeCom、设置等更新/状态角标仍可见。
- 展开/折叠动画不应导致内容区闪烁。

#### 6.4.3 响应式规则

- 大屏默认展开。
- 窄屏默认折叠或抽屉式导航。
- 复杂表格页允许用户手动折叠以释放横向空间。

#### 6.4.4 验收标准

- 折叠后内容区宽度实际增加约 136-144px。
- 所有现有路由仍可正常跳转。
- 当前激活菜单、徽标、权限控制与现有逻辑一致。

### 6.5 WeCom 一键诊断与交互式排错指引

#### 6.5.1 入口

- 在 WeCom 页面头部状态区增加“ 一键诊断 ”按钮。
- 在配置页连接测试附近增加次级入口。
- 诊断运行时展示进度步骤，避免用户误以为卡死。

#### 6.5.2 诊断项

诊断应至少包含:

1. 基础配置
   - `corp_id` 是否存在
   - `agent_id` 是否存在
   - `secret` 是否存在但不明文返回
   - 回调/通知相关配置是否完整
2. PHP 运行环境
   - PHP 版本
   - `curl` 扩展
   - `openssl` 扩展
   - `json` 扩展
   - `pdo/sqlite` 或当前数据库驱动
3. 文件与目录权限
   - `config/wecom.php` 是否可读/可写
   - token 缓存目录是否可写
   - WeCom 日志目录是否可写
4. 网络连通性
   - DNS 解析 `qyapi.weixin.qq.com`
   - HTTPS 连接
   - 获取 access_token
   - 企业微信 API 错误码解析
5. 数据库与表
   - WeCom 相关表是否存在
   - 关键表记录数
   - 最近同步时间
6. 同步与通知
   - 最近同步任务状态
   - 失败日志摘要
   - 通知规则数量与启用状态

#### 6.5.3 结果展示

- 总体状态分为:
  - 通过
  - 警告
  - 失败
  - 未检测
- 每个诊断项包含:
  - 检测名称
  - 状态
  - 检测结果摘要
  - 失败原因
  - 推荐操作
  - 技术详情，可折叠
  - 复制诊断信息按钮
- 对常见问题提供交互式指引:
  - 缺少 PHP 扩展: 展示安装建议与重启 PHP 服务提示。
  - 网络不可达: 展示代理、防火墙、DNS、证书检查建议。
  - 企业微信凭证错误: 指向基础配置页对应字段。
  - 数据表缺失: 指向升级/迁移文档或提供只读检查结果。

#### 6.5.4 后端接口建议

- 新增接口: `api.php?c=WeComAdmin&a=RunDiagnostics&t=web`
- 请求:

```json
{
  "scope": "all",
  "include_sensitive": false
}
```

- 响应:

```json
{
  "overall_status": "warning",
  "summary": {
    "passed": 8,
    "warning": 2,
    "failed": 1,
    "unchecked": 0
  },
  "checks": [
    {
      "key": "php_extension_curl",
      "group": "PHP运行环境",
      "name": "curl 扩展",
      "status": "passed",
      "message": "curl 扩展已启用",
      "suggestion": "",
      "details": {
        "version": "7.x"
      }
    }
  ],
  "generated_at": "2026-04-27 10:00:00"
}
```

#### 6.5.5 验收标准

- 点击“一键诊断”后能展示完整诊断结果，不只显示单一成功/失败。
- 至少覆盖 PHP 扩展、配置完整性、网络连通性、数据库表、企业微信 API 五类问题。
- 失败项必须给出面向管理员的下一步操作。
- secret、token 等敏感信息不得明文返回或展示。

## 7. 非功能需求

- 性能: 仓库联想搜索不得阻塞主表格操作，远程请求需防抖。
- 安全: WeCom 诊断结果不得泄露 `secret`、access_token、完整敏感请求头。
- 兼容: 保持 Vue 2 + iView/View UI 现有组件体系。
- 可维护: 诊断项后端输出结构化数据，前端按通用组件渲染，便于后续扩展。
- 可访问性: 侧边栏折叠、联想搜索、诊断面板支持键盘基础操作。

## 8. 数据与埋点建议

- 仓库搜索:
  - 搜索关键词长度
  - 候选点击率
  - 搜索后定位成功率
- 布局:
  - 侧边栏折叠使用率
  - 表格页折叠触发次数
- 后台任务:
  - 悬浮球点击率
  - 拖拽/最小化使用率
- WeCom 诊断:
  - 诊断运行次数
  - 失败项分布
  - 诊断后连接测试成功率

## 9. 优先级与里程碑

### P0

- WeCom 一键诊断后端结构化接口。
- WeCom 诊断结果面板与排错指引。
- 侧边栏折叠基础能力。

### P1

- 仓库页 Search-as-you-type 联想搜索。
- 权限路径层级展示优化。
- `global-task-ball` 默认位置、视觉样式和运行状态优化。

### P2

- 悬浮球拖拽位置记忆。
- 诊断结果复制/导出。
- 搜索命中行高亮与跨页定位细节优化。

## 10. 风险与依赖

- 权限路径层级化依赖现有 authz 解析结果质量，若当前接口只返回平铺路径，需要前端构建路径树或补充后端结构化返回。
- WeCom 网络诊断可能受服务器网络策略影响，应区分“服务器不可达”和“企业微信 API 鉴权失败”。
- PHP 扩展检测需要兼容容器化部署和传统部署，建议仅检测运行时事实，不直接执行安装。
- 侧边栏折叠会影响所有页面，需要重点回归宽表格、弹窗、菜单徽标和移动端。

## 11. 开放问题

1. 仓库联想搜索是否需要新增服务端接口，还是先基于现有列表接口和本地缓存实现。
2. 权限路径层级是否只覆盖仓库权限弹窗，还是同时覆盖用户权限查看弹窗。
3. WeCom 诊断是否允许提供“自动创建缺失表”等修复按钮。本 PRD 建议一期只做诊断与指引。
4. 悬浮球拖拽位置是否按用户维度保存到后端，还是仅保存到浏览器本地。
