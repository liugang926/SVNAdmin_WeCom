# 表格列宽可拖拽调整功能

## 功能概述

为SVNAdmin系统的用户、分组、仓库列表页面添加了表格列宽可拖拽调整功能，用户可以根据需要自定义调整列宽以便更好地查看内容。

## 功能特性

### 1. 可拖拽调整列宽
- 鼠标悬停在表格列头边界时，光标会变为调整大小的样式
- 拖拽列边界可以实时调整列宽
- 支持所有可调整的列（序号和操作列除外）

### 2. 本地存储偏好设置
- 用户调整的列宽会自动保存到浏览器本地存储
- 下次访问页面时会自动恢复用户的列宽设置
- 每个页面的列宽设置独立存储

### 3. 智能列宽配置
- 重要内容列（如备注信息）默认宽度更大（300px）
- 数值类列（如用户数、分组数）使用适中宽度（100-140px）
- 固定功能列（序号、操作）不可调整，保持界面稳定

## 涉及页面

### 1. SVN分组列表 (`/repositoryGroup`)
- 存储键：`svn_group_column_widths`
- 可调整列：分组名、包含用户数、包含分组数、包含别名用户数、备注信息

### 2. SVN用户列表 (`/repositoryUser`)
- 存储键：`svn_user_column_widths`
- 可调整列：用户名、密码、启用状态、备注信息、有权路径、上次登录、在线状态

### 3. SVN仓库列表 (`/repositoryInfo`)
- 存储键：`svn_repository_column_widths`
- 可调整列：仓库名、版本数、体积、备注信息、仓库内容、仓库权限、仓库钩子

## 技术实现

### 1. 前端实现
- 使用iView UI框架的Table组件的`resizable`属性
- 监听`@on-column-width-resize`事件保存列宽设置
- 在组件挂载时从localStorage加载保存的列宽设置

### 2. 列配置更新
```javascript
// 原配置
{
  title: "备注信息",
  slot: "svn_group_note",
  minWidth: 120,
}

// 新配置
{
  title: "备注信息",
  slot: "svn_group_note",
  width: 300,
  resizable: true,
}
```

### 3. 本地存储结构
```javascript
// localStorage中的数据格式
{
  "svn_group_name": 180,
  "include_user_count": 150,
  "svn_group_note": 400
}
```

## 用户体验改进

### 解决的问题
1. **内容显示不完整**：特别是备注信息列经常被截断
2. **列宽固定**：无法根据实际内容调整合适的显示宽度
3. **用户体验差**：需要悬停或点击才能看到完整内容

### 改进效果
1. **灵活调整**：用户可以根据需要调整任意列的宽度
2. **个性化设置**：每个用户的列宽偏好独立保存
3. **即时生效**：拖拽调整后立即生效并保存
4. **持久化存储**：设置在浏览器中持久保存

## 使用方法

1. **调整列宽**：
   - 将鼠标移动到表格列头的边界处
   - 当光标变为调整大小样式时，按住鼠标左键拖拽
   - 释放鼠标完成调整

2. **重置列宽**：
   - 清除浏览器本地存储中对应的键值
   - 刷新页面恢复默认列宽

## 兼容性

- 支持现代浏览器（Chrome、Firefox、Safari、Edge）
- 依赖localStorage API
- 兼容iView UI框架的Table组件

## 注意事项

1. 列宽设置存储在浏览器本地，清除浏览器数据会丢失设置
2. 不同浏览器的设置不会同步
3. 序号列和操作列不可调整，保持界面一致性
